üìù Expressions

Expression PDC Tag

pdc tag %string% of %objects%

Description: Gets or sets a PDC tag value with automatic type detection. Supports GET, SET, DELETE, ADD, and REMOVE operations.

Supported Objects: Items (ItemStack, ItemType, Slot), Entities (all types), Chunks

Examples
# Set different data types set pdc tag "myplugin:level" of player's tool to 5 set pdc tag "myplugin:name" of player's tool to "Legendary Sword" set pdc tag "myplugin:damage" of player's tool to 12.5 # Get values set {_level} to pdc tag "myplugin:level" of player's tool send "Item level: %{_level}%" # Delete tag delete pdc tag "myplugin:level" of player's tool # Math operations add 1 to pdc tag "myplugin:kills" of player remove 10 from pdc tag "myplugin:mana" of player # Chunk support set pdc tag "region:protected" of chunk at player to true

Expression All PDC Tags

[all] [the] pdc tag[s] of %objects%

Description: Returns a list of all PDC tag keys present on an object.

Examples
# Get all tags from item set {_tags::*} to all pdc tags of player's tool send "This item has %size of {_tags::*}% tags" # Loop through all tags loop all pdc tags of player's tool: set {_value} to pdc tag loop-value of player's tool send "%loop-value%: %{_value}%" # Get chunk tags set {_chunkTags::*} to pdc tags of chunk at player loop {_chunkTags::*}: send "Chunk tag: %loop-value%"

Effect Clear PDC

clear [all] pdc (of|from) %objects%

Description: Removes all PDC tags from an object.

Examples
# Clear all tags from item clear all pdc of player's tool # Clear entity data clear pdc from last spawned entity # Clear chunk data clear pdc of chunk at player

Effect Remove PDC Tags

(remove|delete) pdc tag[s] %strings% (of|from) %objects%

Description: Removes one or more specific PDC tags from an object.

Examples
# Remove single tag remove pdc tag "myplugin:temp" of player's tool # Remove chunk tag delete pdc tag "region:owner" of chunk at player # Remove multiple tags using list set {_tags::*} to "myplugin:level", "myplugin:exp" and "myplugin:kills" remove pdc tags {_tags::*} of player's tool

Complex Data Types

For any other Skript data type (like location, itemstack, etc.), SkriptPDC uses Skript's serialization system to store and retrieve the data.

# Storing a location set pdc tag "homes:spawn" of player to player's location # Retrieving and using the location set {_home} to pdc tag "homes:spawn" of player teleport player to {_home}
Important: When storing complex objects like players, it's recommended to store their UUID as text instead of the player object itself, as player objects are temporary snapshots.

‚ùì FAQ & Troubleshooting

How do I use namespaced keys?

PDC tags should follow the format namespace:key. If you omit the namespace, SkriptPDC will automatically use your plugin's namespace. For example:

# These are equivalent: set pdc tag "level" of player's tool to 5 set pdc tag "skriptpdc:level" of player's tool to 5 # Use custom namespace: set pdc tag "myplugin:level" of player's tool to 5

Can I store data on chunks?

Yes! Version 3.0 adds full support for storing PDC data on chunks. This is perfect for region protection, claim systems, or world management features.

# Store data on chunk set pdc tag "region:name" of chunk at player to "Spawn Area" # Retrieve chunk data set {_name} to pdc tag "region:name" of chunk at player

Why isn't my item data saving?

When working with items in variables or specific slots, you must update the slot after modifying PDC data:

# Incorrect - changes won't save: set {_item} to slot 0 of player's inventory set pdc tag "mydata" of {_item} to "value" # Correct - update the slot: set {_item} to slot 0 of player's inventory set pdc tag "mydata" of {_item} to "value" set slot 0 of player's inventory to {_item} # Using player's tool handles this automatically: set pdc tag "mydata" of player's tool to "value"

What objects support PDC?

SkriptPDC supports all objects that implement Bukkit's PersistentDataHolder interface:

  • Items: ItemStack, ItemType, Inventory Slots
  • Entities: All entity types (Players, Mobs, Armor Stands, etc.)
  • Chunks: World chunks for region-based data

How do I check if a tag exists?

You can check by attempting to retrieve the tag and testing if it's set:

# Check if tag exists set {_value} to pdc tag "mydata" of player's tool if {_value} is set: send "Tag exists with value: %{_value}%" else: send "Tag does not exist" # Alternative using otherwise set {_value} to pdc tag "mydata" of player's tool otherwise "default"

Can I store lists or multiple values?

PDC stores single values per tag. For lists, you can either:

  • Use multiple tags: myplugin:value1, myplugin:value2, etc.
  • Serialize a list as text (join with delimiter)
  • Store complex objects using Skript's serialization
# Example: Multiple tags approach set pdc tag "homes:home1" of player to player's location set pdc tag "homes:home2" of player to location(100, 64, 200, world "world") # Example: Serialized list set {_friends::*} to "Player1", "Player2" and "Player3" set pdc tag "friends:list" of player to {_friends::*}
